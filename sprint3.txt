Plan du sprint 3
Nous pensons dans ce sprint finir par la partie blazor en entier, les tâches précise que nous devons réaliser sont :
-La page modification des utilisateurs
-le bouton permettant de supprimer les utilisateurs (avec un pop-up)
-l’utilisation d’une API (Application Programming Interface)
-l’utilisation de DI (Dependency Injection)
-l’utilisation d’IOC (Inversion Of Control)
-La possibilité d’utilisé plusieurs langue pour le site (au moins 2)
-L’optimisation du code avec sonarqube
-GUI (Graphique User Interface)
-Rendre le code non visible depuis l’interface utilisateur

Nous devons aussi faire toute la documentation du blazor

Ensuite nous refaire la gestion du vocabulaire en blazor , car le langage est plus pratique pour cela.
